# Permission Admin Template

Purpose: role, permission, menu, data scope, or user authorization management.

Use when the PRD mentions role, permission, menu, authorization, data scope, or access control.

Required structure:
- Role/user list.
- Permission tree or grouped permission matrix.
- Data scope or operation permission area.
- Save and reset actions.

Interaction requirements:
- Role selection should update visible detail state.
- Permission toggles should look clickable.
- Save button should show local success feedback.
