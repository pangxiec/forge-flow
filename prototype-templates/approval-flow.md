# Approval Flow Template

Purpose: review, approve, reject, return, or transfer workflow items.

Use when the PRD mentions approval, review, pass, reject, workflow, state transition, or audit.

Required structure:
- Current node and status.
- Applicant or initiator summary.
- Approval comments input.
- Action buttons for approve and reject.
- Timeline showing submitted, reviewed, approved/rejected states.

Interaction requirements:
- Approve and reject must be clickable.
- Action should update UI state or navigate to next page.
- Rejection should expose reason/comment field.
