# Searchable List Template

Purpose: list, query, filter, compare, and enter detail.

Use when the PRD mentions list, query, search, filter, pagination, batch operation, or record management.

Required structure:
- Filter area with keyword, status, owner, time range, or domain-specific filters.
- Data table with business-specific columns.
- Status tags and action buttons.
- Empty/loading/error states as static UI examples.

Interaction requirements:
- Search/filter button should update visible state or show feedback.
- Row and view button should enter detail.
- New button should enter create/edit form.
