# Detail View Template

Purpose: inspect one business object and perform contextual actions.

Use when the PRD mentions detail, view, record, audit trail, attachment, or field explanation.

Required structure:
- Header with title, status, primary action, and back button.
- Key information grid.
- Detail sections for fields, rules, attachments, timeline, and comments.
- Related records or processing history when useful.

Interaction requirements:
- Back button returns to list.
- Primary action enters approval, edit, or next workflow state.
- Tabs or section switches should work locally if present.
