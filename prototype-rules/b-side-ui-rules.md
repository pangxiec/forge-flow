# B-side UI Rules

- Build the actual product interface as the first screen, not a landing page.
- Prefer quiet, utilitarian, information-dense layouts.
- Use tables, filters, forms, status tags, timelines, metric cards, and detail sections.
- Keep card radius 8px or less.
- Avoid marketing hero sections, oversized slogans, decorative illustrations, gradient blobs, and one-note palettes.
- Use business-specific fields and actions from the PRD.
- Text must fit inside buttons, cards, and table cells.
