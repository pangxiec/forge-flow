# 数据库表设计规范

## 1. 适用范围

本文档用于规范项目数据库表、字段、主键、外键和通用字段的设计方式，作为 AI 生成数据库设计、后端代码结构和数据访问层代码时的基础知识库规范。

## 2. 命名规范

### 2.1 表命名

所有数据库表名统一使用小写下划线命名。

示例：

```sql
project
requirement_version
approval_record
git_repository
```

不推荐：

```sql
Project
requirementVersion
approval-record
```

### 2.2 字段命名

所有字段名统一使用小写下划线命名。

示例：

```sql
project_name
created_at
updated_by
approval_status
```

不推荐：

```sql
projectName
CreatedAt
updated-by
```

## 3. 主键规范

所有表的主键统一命名为 `id`。

主键类型建议使用 `BIGINT`。

示例：

```sql
id BIGINT PRIMARY KEY COMMENT '主键ID'
```

## 4. 外键规范

不建议使用数据库物理外键。

表之间的关联关系统一由业务层保证，包括：

- 数据写入时的关联合法性校验。
- 数据删除时的引用检查。
- 查询时通过业务代码或 ORM 显式关联。
- 需要级联处理时，由业务服务控制处理逻辑。

示例：

```sql
project_id BIGINT NOT NULL COMMENT '项目ID'
created_by BIGINT NULL COMMENT '创建人ID'
```

不推荐：

```sql
CONSTRAINT fk_project_id FOREIGN KEY (project_id) REFERENCES project(id)
```

## 5. 通用字段规范

所有核心业务表必须包含以下通用字段：

```sql
id BIGINT PRIMARY KEY COMMENT '主键ID',
created_at DATETIME NOT NULL COMMENT '创建时间',
updated_at DATETIME NOT NULL COMMENT '更新时间',
created_by BIGINT NULL COMMENT '创建人ID',
updated_by BIGINT NULL COMMENT '更新人ID',
deleted TINYINT NOT NULL DEFAULT 0 COMMENT '是否删除：0否，1是',
version INT NOT NULL DEFAULT 1 COMMENT '数据版本号'
```

字段说明：

| 字段名 | 类型 | 是否必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| id | BIGINT | 是 | 无 | 主键ID |
| created_at | DATETIME | 是 | 无 | 创建时间 |
| updated_at | DATETIME | 是 | 无 | 更新时间 |
| created_by | BIGINT | 否 | NULL | 创建人ID |
| updated_by | BIGINT | 否 | NULL | 更新人ID |
| deleted | TINYINT | 是 | 0 | 是否删除：0否，1是 |
| version | INT | 是 | 1 | 数据版本号 |

## 6. 建表示例

```sql
CREATE TABLE project (
  id BIGINT PRIMARY KEY COMMENT '主键ID',
  project_name VARCHAR(128) NOT NULL COMMENT '项目名称',
  project_code VARCHAR(64) NOT NULL COMMENT '项目编码',
  description VARCHAR(512) NULL COMMENT '项目描述',
  status VARCHAR(32) NOT NULL COMMENT '项目状态',
  created_at DATETIME NOT NULL COMMENT '创建时间',
  updated_at DATETIME NOT NULL COMMENT '更新时间',
  created_by BIGINT NULL COMMENT '创建人ID',
  updated_by BIGINT NULL COMMENT '更新人ID',
  deleted TINYINT NOT NULL DEFAULT 0 COMMENT '是否删除：0否，1是',
  version INT NOT NULL DEFAULT 1 COMMENT '数据版本号'
) COMMENT='项目表';
```

## 7. AI 生成要求

AI 生成数据库设计时必须遵守以下规则：

1. 表名必须使用小写下划线。
2. 字段名必须使用小写下划线。
3. 所有表主键统一命名为 `id`。
4. 主键类型建议使用 `BIGINT`。
5. 不生成数据库物理外键。
6. 核心业务表必须包含通用字段。
7. 关联字段使用 `{业务对象}_id` 命名，例如 `project_id`、`prd_id`、`approval_task_id`。
8. 字段必须补充清晰的 `COMMENT` 说明。
