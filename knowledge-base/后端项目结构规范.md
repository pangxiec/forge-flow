## 2. 后端根目录结构

采用 Maven 多模块架构，按职责拆分模块，依赖单向流转：

```
project-name/
├── pom.xml                          # 父 POM，统一依赖版本管理
├── project-name-common/             # 公共模块：常量、枚举、注解、工具、上下文、拦截器、结果封装
├── project-name-pojo/               # 实体传输模块：VO（请求/响应）、DTO（输入/输出）
├── project-name-dao/                # 数据访问模块：Domain 实体、Mapper 接口、Mapper XML
├── project-name-third/              # 第三方集成模块：Forest/HTTP 客户端、外部 API 封装
├── project-name-message/            # 消息模块：邮件、企业微信、短信等消息通道
└── project-name-api-admin/          # 主服务模块：启动类、Controller、Service、Config、定时任务
```

### 2.1 模块依赖关系（单向，禁止循环）

```
common  ←  pojo  ←  dao
                   ←  third
                   ←  message
                        ←  api-admin（聚合入口）
```

- `common`：最底层，不依赖其他业务模块
- `pojo`：依赖 `common`
- `dao` / `third` / `message`：依赖 `pojo`
- `api-admin`：依赖 `dao`、`third`、`message`，是唯一可启动模块

## 3. 各模块目录与包结构规范

### 4.2 Controller 层规范

```java
@Api(tags = "黑名单")                              // Knife4j 接口分组
@Log4j
@RestController
@RequestMapping("/api/v1/black")                  // 统一 /api/v1/ 前缀
@EnableResponseResult                             // 开启统一响应包装
public class BlackListController {

    @Resource
    private BlackListService blackListService;

    @PostMapping("/addBlackUser")
    @ApiOperation(value = "添加黑名单用户")
    @OptionalLog(module = "黑名单", optType = OptTypeEnum.CREATE)  // 操作日志
    public Boolean addBlackUser(@RequestBody @Valid ReqAddBlackUserVo reqVo) {
        return blackListService.addBlackUserInfo(reqVo);
    }

    @GetMapping("/getBlackUserList")
    @ApiOperation(value = "获取黑名单列表")
    public PageResult<RespBlackUserListVo> getBlackUserList(ReqBlackUserLstVo reqVo) {
        return blackListService.getBlackUserPage(reqVo);
    }
}
```

**要点**：

- Controller 不写业务逻辑，只做参数接收、委托调用
- 使用 `@EnableResponseResult` 注解自动包装统一响应
- 使用 `@Valid` 校验请求参数
- RESTful 风格：`@GetMapping` 查询、`@PostMapping` 新增、`@PutMapping` 修改、`@DeleteMapping` 删除
- URL 统一 `/api/v1/模块名/操作` 格式

### 4.3 Service 层规范

**接口**：

```java
public interface BlackListService extends IService<BlackList> {

    /**
     * 新增黑名单用户
     *
     * @param dto 入参
     * @return 是否成功
     * @author xxx
     */
    Boolean addBlackUserInfo(ReqAddBlackUserVo vo);
}
```

**实现**：

```java
@Log4j2
@Service
public class BlackListServiceImpl extends ServiceImpl<BlackListMapper, BlackList>
        implements BlackListService {

    @Resource
    private BlackListMapper blackListMapper;

    @Override
    public Boolean addBlackUserInfo(ReqAddBlackUserVo vo) {
        // 业务校验
        int count = countByType(dto.getUserId(), dto.getServiceType());
        if (count > 0) {
            throw new BizException("不可重复添加！");
        }

        // 构建实体
        BlackList entity = new BlackList();
        entity.setUserId(dto.getUserId());
        entity.setServiceType(dto.getServiceType());

        // 持久化
        int result = blackListMapper.insert(entity);
        return result > 0;
    }

    @Override
    public PageResult<RespBlackUserListVo> getBlackUserPage(InputBlackUserLstDto dto) {
        Page<BlackList> page = new Page<>(dto.getPage(), dto.getLimit());
        Page<BlackList> result = this.page(page,
                Wrappers.<BlackList>lambdaQuery()
                        .like(StrUtil.isNotBlank(dto.getUserId()), BlackList::getUserId, dto.getUserId())
                        .eq(dto.getServiceType() != null, BlackList::getServiceType, dto.getServiceType())
                        .orderByDesc(BlackList::getModified)
        );
        // 转换并封装分页结果
        List<RespBlackUserListVo> items = BeanHelper.convertBeans(result.getRecords(), RespBlackUserListVo::new);
        PageResult<RespBlackUserListVo> pageResult = new PageResult<>();
        pageResult.setItems(items);
        pageResult.setTotal(page.getTotal());
        return pageResult;
    }
}
```

**要点**：

- 接口继承 `IService<T>`，实现类继承 `ServiceImpl<Mapper, Entity>`
- 使用 `@Resource` 注入（统一风格）
- 业务异常使用 `BizException`（全局捕获）
- 查询优先使用 MyBatis Plus 的 `Wrappers.lambdaQuery()`，复杂 SQL 写 XML
- Service 返回领域结果（DTO/VO/Entity），不返回 HTTP 相关对象

### 4.4 Domain 实体规范

```java
@Data
@TableName("t_black_list")
public class BlackList {

    private static final long serialVersionUID = 1L;

    @TableId(value = "id", type = IdType.ASSIGN_ID)
    private Long id;

    @TableField("user_id")
    private String userId;

    @TableField("type")
    private BlackUserTypeEnum type;          // 枚举类型字段

    @TableField(value = "created_at", fill = FieldFill.INSERT)
    private Date createdAt;                    // 创建时间（自动填充）

    @TableField(value = "updated_at", fill = FieldFill.INSERT_UPDATE)
    private Date updatedAt;                   // 修改时间（自动填充）
}
```

**要点**：

- 表名使用 `@TableName` 显式映射
- 主键使用 `@TableId(type = IdType.ASSIGN_ID)`
- 字段使用 `@TableField` 显式映射列名
- 审计字段（`created``at`_/`modified`_at）使用 `FieldFill` 自动填充
- 枚举类型字段直接使用枚举类（配合 MyBatis Plus 枚举处理）

### 4.9 统一响应规范

```java
@Data
public class ResponseResult<T> implements Serializable {

    private static final long serialVersionUID = 1L;
    private String code;
    private Boolean success;
    private String msg;
    private T data;

    public static <T> ResponseResult<T> success(T data) {
        ResponseResult<T> result = new ResponseResult<>();
        result.setCode(ResponseCodeEnum.SUCCESS.getCode());
        result.setMsg(ResponseCodeEnum.SUCCESS.getMsg());
        result.setSuccess(true);
        result.setData(data);
        return result;
    }

    public static ResponseResult<Boolean> fail(String msg) {
        ResponseResult<Boolean> result = new ResponseResult<>();
        result.setCode(ResponseCodeEnum.BUSINESS_DEFAULT_ERROR.getCode());
        result.setMsg(msg);
        result.setSuccess(false);
        return result;
    }
}
```

```java
// 通过 @ControllerAdvice + ResponseBodyAdvice 自动包装
@ControllerAdvice(annotations = EnableResponseResult.class)
public class ResponseResultHandler implements ResponseBodyAdvice {

    @Override
    public boolean supports(MethodParameter methodParameter, Class converterType) {
        return true;
    }

    @Override
    public Object beforeBodyWrite(Object body, MethodParameter methodParameter,
                                  MediaType mediaType, Class converterType,
                                  ServerHttpRequest request, ServerHttpResponse response) {
        // 方法标注 @IgnoreResponseResult 则跳过包装
        boolean ignore = methodParameter.getMethodAnnotation(IgnoreResponseResult.class) != null;
        if (ignore) {
            return body;
        }
        // 已经是 ResponseResult 则直接返回
        if (body instanceof ResponseResult) {
            return body;
        }
        // 包装为统一响应
        ResponseResult<Object> result = ResponseResult.success(body);
        if (body instanceof String) {
            return JsonUtil.toJson(result);
        }
        return result;
    }
}
```

**要点**：Controller 标注 `@EnableResponseResult`，返回原始数据，由全局 Handler 自动包装为 `ResponseResult`。

## 6. 数据库规范

- 表名：`t_` 前缀 + 下划线命名，如 `t_black_list`
- 字段名：下划线命名，如 `user_id`、`created`、`modified`
- 每张表必须包含：`id`（主键自增）、`created`（创建时间）、`modified`（修改时间）
- 审计扩展字段（可选）：`create_user_id`、`create_user_name`、`update_user_id`、`update_user_name`
- 状态字段使用整型枚举值，配合 Java 枚举映射

## 7. 分层调用规范

```
前端请求
  → Controller（@Valid 校验 → VO 转 DTO）
    → Service（业务逻辑 → MyBatis Plus 操作 Entity）
      → Mapper / Mapper.xml（数据持久化）
    → Manager（跨 Service 编排，@Transactional 事务边界）
    → Third Service（Forest 调用外部 API）
  → ResponseResultHandler（统一响应包装）
前端响应
```

**禁止事项**：

- Controller 直接调用 Mapper
- Service 返回 HTTP 相关对象（HttpServletRequest 等）
- 跨模块循环依赖

## 8. 命名规范汇总

| 类型         | 命名规则                     | 示例                      |
| ---------- | ------------------------ | ----------------------- |
| Controller | `业务名` + `Controller`     | `BlackListController`   |
| Service 接口 | `业务名` + `Service`        | `BlackListService`      |
| Service 实现 | `业务名` + `ServiceImpl`    | `BlackListServiceImpl`  |
| Mapper     | `业务名` + `Mapper`         | `BlackListMapper`       |
| Entity     | 实体名（无后缀）                 | `BlackList`             |
| 请求 VO      | `Req` + `业务名` + `Vo`     | `ReqAddBlackUserVo`     |
| 响应 VO      | `Resp` + `业务名` + `Vo`    | `RespBlackUserListVo`   |
| 输入 DTO     | `Input` + `业务名` + `Dto`  | `InputAddBlackUserDto`  |
| 输出 DTO     | `Output` + `业务名` + `Dto` | `OutputCallTaskListDto` |
| 枚举         | `业务名` + `Enum`           | `CallStatusEnum`        |
| 常量         | `业务名` + `Constant`       | `RedisKeyConstant`      |
| 配置类        | `技术名` + `Config`         | `MybatisPlusConfig`     |
| 第三方接口      | `I` + `业务名` + `Service`  | `IMebMessageService`    |
| Manager    | `业务名` + `Manager`        | `DetailManager`         |
| 切面         | `功能名` + `Aspect`         | `TimedTaskLockAspect`   |
| 监听器        | `业务名` + `Listener`       | `TaskSystemListener`    |
| 定时任务       | `业务名` + `Task`           | `CallTask`              |

## 9. 注解使用规范

| 注解                       | 作用范围            | 说明               |
| ------------------------ | --------------- | ---------------- |
| `@EnableResponseResult`  | Controller 类/方法 | 开启统一响应包装         |
| `@IgnoreResponseResult`  | Controller 方法   | 跳过响应包装（如文件下载）    |
| `@EnableAuth`            | Controller 类    | 开启鉴权（配合 SaToken） |
| `@IgnoreAuth`            | Controller 方法   | 跳过鉴权（如登录接口）      |
| `@OptionalLog`           | Controller 方法   | 操作日志记录           |
| `@TimedTaskLock`         | 定时任务方法          | Redis 分布式锁防止重复执行 |
| `@ApiModelProperty`      | VO 字段           | 接口文档字段说明         |
| `@NotBlank` / `@NotNull` | VO 字段           | 参数校验             |
| `@TableName`             | Entity 类        | 映射数据库表名          |
| `@TableId`               | Entity 字段       | 映射主键             |
| `@TableField`            | Entity 字段       | 映射列名、自动填充策略      |
